<Callout type="warning" title="Draft">
This page documents the cryptographic contract between your device and the replication
endpoint. The short version: **the server only ever holds AES-256-GCM ciphertext and
SHA-256 path hashes.** It cannot read records, cannot enumerate key names, and cannot
be coerced into doing either.
</Callout>

## What the server sees vs. cannot see

```
                      ┌─────────────────────────────────┐
                      │  Replication server's view      │
                      ├─────────────────────────────────┤
SEES         ►        │  • ciphertext bytes             │
                      │  • SHA-256(path) — opaque hash  │
                      │  • project_id (auth scope)      │
                      │  • content length               │
                      │  • timestamp                    │
                      ├─────────────────────────────────┤
CANNOT SEE   ►        │  • plaintext content            │
                      │  • path / key name              │
                      │  • embedding vector             │
                      │  • the encryption key           │
                      │  • metadata fields              │
                      └─────────────────────────────────┘
```
