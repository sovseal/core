<Callout type="info">
sovseal is built around a single architectural commitment: **plaintext never leaves the
device, and reads never touch the network.** Every other guarantee follows from that.
</Callout>

## The shape of a sovseal deployment

```
┌─────────────────────────── Agent Process ───────────────────────────┐
│                                                                     │
│   ┌──────────────┐   store    ┌────────────────────────────┐        │
│   │  Your agent  │ ─────────► │  sovseal SDK / MCP server  │        │
│   │  (LLM tool   │            │  ───────────────────────── │        │
│   │   calls)     │ ◄───────── │  • LanceDB (vectors)       │        │
│   └──────────────┘   recall   │  • Transformers.js (384-d) │        │
│                               │  • AES-256-GCM (per record)│        │
│                               └────────────┬───────────────┘        │
│                                            │ write-behind           │
│                                            │ (ciphertext only)      │
└────────────────────────────────────────────┼────────────────────────┘
                                             ▼
                              ┌────────────────────────────┐
                              │  Replication endpoint      │
                              │  (Platform or self-hosted) │
                              │  ───────────────────────── │
                              │  Sees: ciphertext + SHA-256│
                              │  path hashes. Nothing else.│
                              └────────────────────────────┘
```
